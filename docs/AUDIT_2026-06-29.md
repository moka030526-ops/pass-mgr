# Deep Audit — 2026-06-29

A multi-agent, adversarially-verified bug hunt and security review across the **entire**
codebase (core, desktop, ffi), run in iterative rounds. Each candidate finding was confirmed
by ≥2 independent verifier agents under a stated threat model before being acted on.

**Threat model (security):** a LOCAL encrypted password/asset vault. In scope: theft of the
on-disk vault files (confidentiality + tamper/rollback detection), a crafted vault /
other-vault / document fed to open/merge/extract/migrate, malicious filenames/paths/content,
and other local processes/users. Out of scope: an attacker already running as the user with
the vault unlocked in RAM.

Result: **no Critical or High** findings. The vault's core cryptography, atomic-write
discipline, and input sanitization are strong. The actionable findings were a symlink-follow
gap on the export/extract *write* path, a CSV formula-injection gap, an O(N²) read scan, and
a few defense-in-depth items.

## Security review — round 1

| ID | Sev | Area | Status |
|----|-----|------|--------|
| M1 | Medium | Export/extract followed symlinked **intermediate** dirs → decrypted plaintext could escape the export root | **Fixed** |
| M2 | Medium | CSV export did not neutralize spreadsheet formula triggers (`= + - @`) → formula injection / password exfiltration in a spreadsheet | **Fixed** |
| M3 | Medium (availability) | `VolumeStore::read` did a linear entry scan per call → bulk read paths (export/compact/migrate/merge) O(N²) | **Fixed** |
| L1 | Low | Redundancy recovery derives up to `MAX_RECOVERY_SALTS`×2 maxed-Argon2 keys on a failed open (CPU/battery burn) | Open (planned) |
| L2 | Low | `read_file_capped` let `read_to_end` realloc, stranding cleartext doc fragments in freed heap | **Fixed** |
| L3 | Low | No per-partition manifest **entry-count** cap (amplifies M3; import/merge O(M²)) | Open (planned) |
| — | Likely | Lock file opened without `O_NOFOLLOW` (the one attacker-reachable open that still follows symlinks) | **Fixed** |
| — | Likely/accepted | Manifest-only rollback (manifest AAD omits `seq`) — already a documented at-rest limitation; record ids change per version so `referenced ⊆ stored` catches the realistic case | Accepted; optional DiD |

### Fixes applied this round

- **M1 — symlink-hardened export.** Added `vault::reject_symlinked_descendants(root, leaf)`
  and call it before `create_dir_all` in `export_document_into` (`vault.rs`) and `cli_extract`
  (`main.rs`). Any pre-planted symlink among the intermediate directories of the (user-chosen,
  reused) export root is now rejected — mirroring the import-side `reject_symlink_dir`
  discipline — so decrypted plaintext can't be redirected outside the root, and the
  `harden_dir` chmod no longer runs through an unverified intermediate. Regression test:
  `export_document_into_rejects_symlinked_intermediate`.
- **M2 — CSV anti-formula prefixing.** `csv::esc` now prefixes a cell that begins with
  `= + - @` with a single quote so spreadsheets render it as text. Realistic passwords (no
  leading trigger) round-trip unchanged. Test: `esc_neutralizes_leading_formula_triggers`.
- **M3 — O(1) `entry()`.** `Located` now carries `entry_index`; `reindex` populates it and
  `VolumeStore::entry` indexes directly (with a safe fallback scan on any desync), removing
  the per-read linear scan. Bulk read paths drop from O(N²) to O(N log N).
- **L2 — pre-sized Zeroizing read buffer.** `read_file_capped` now pre-sizes a `Zeroizing`
  buffer to the file's stat'd length (clamped to the ceiling) so `read_to_end` never
  reallocates and strands cleartext, and returns `Zeroizing<Vec<u8>>` directly.
- **Lock `O_NOFOLLOW`.** `WriteLock::acquire` now opens the lock file with `O_NOFOLLOW`
  (Unix), surfacing a symlinked lock path as `Locked` (via `ELOOP`) instead of following it —
  consistent with `single_instance.rs` and the volume writers; still no chmod on the lock.

### Planned (later rounds)
- **L1**: lower `MAX_RECOVERY_SALTS` and/or clamp recovery KDF params to a recovery floor.
- **L3**: enforce a per-partition entry-count ceiling in `load_manifest` and the import path.
- **Write-side O(N²)**: `staged_rewrite`/import re-serialize the whole partition manifest per
  `put`, so bulk *rewrites* remain O(N²) (availability-only, Ctrl-C-able). Batched-commit
  rewrite is a larger follow-up.

### Notable strengths (verified)
- `VolumeStore::read` cross-checks the AEAD frame's id, path, **and** decrypted size against
  the manifest (AAD binds only `vault_id+partition`); manifests are AEAD-authenticated before
  any `serde_json` parse.
- Rigorous symlink/path hardening almost everywhere (import rejects symlinked dirs; reads use
  `O_NOFOLLOW`/`symlink_metadata`; volume writes use `O_NOFOLLOW`+`create_new`/O_EXCL); M1 was
  the outlier, now closed.
- Conservative, audit-driven Argon2 bounds; deliberate secret zeroization on the write path
  (L2 was the one missed read path, now fixed); atomic rename-replace with parent fsync.

## Bug hunt — round 1

23 confirmed (≥2 verifiers), 5 likely, out of 40 raw (93 agents). No correctness bug is a
security hole; the three High items are data-loss/headline-correctness.

### High — all fixed
| ID | Area | Problem | Status |
|----|------|---------|--------|
| H1 | `gui.rs delete_current` | Failed save left the record removed in memory; a later save silently committed the "failed" deletion (data loss) | **Fixed** |
| H2 | `storage.rs open`/`append_frame` | A surviving manifest with a missing volume opened silently; the next `put` zero-filled and destroyed all prior frames in the partition | **Fixed** |
| H3 | `gui.rs tab_summary` | Grand-total omitted Cash, understating Assets Σ / Net | **Fixed** |

- **H1**: `delete_current` now captures a rollback closure (re-insert the removed record, truncate
  the `remove()` audit entry, restore the edit buffer) and runs it when `persist()` fails.
- **H2**: `VolumeStore::open` fails closed (`Corrupt`) when a manifest loads but its volume file is
  absent; `append_frame` additionally refuses to write at `start > current_len` (no sparse-hole
  zero-fill).
- **H3**: one-line `total.asset_cash += r.asset_cash;` (the TUI was already correct).

### Medium
| ID | Area | Status |
|----|------|--------|
| M4 | `vault.rs export_documents/export_manifests/export_tree` bypass deletion tombstones → a "deleted" secret resurrected into an UNENCRYPTED export | **Fixed** (filter `deleted_docs`) |
| M5 | TUI `attach_document` bypasses Asset owner/numeric-value validation | **Fixed** (validate before commit) |
| M6 | GUI `handle_doc` attach bypasses Asset validation | **Fixed** (`asset_validation_error` gate) |
| M7 | No-echo password prompt treats Ctrl+C/Ctrl+D as literal chars | **Fixed** (Ctrl+C/D cancel; ignore other Ctrl) |
| M8 | `recover_pending_rekey` can lock out read-only opens if `.rekey` removal fails | **Documented** (low likelihood: root-owned/immutable/EIO staging; crash-recovery semantics left intact to avoid regression) |
| M9 | Asset migration drops the description dir → two same-owner+second+name docs share a path | **Accepted by design** — the owner-first scheme deliberately has no description level; docs resolve by id and export de-dups, so no data loss |

### Low — applied where clear, else documented
Fixed: crypto `m_cost ≥ 8·p_cost` in `validate()`; `detail_is_secret` requires a colon (no
over-masking of "…password" titles); `append_frame` `set_len` checked; `highest_partition_index`
rejects non-canonical/leading-zero names; piped-EOF password reads error instead of silently empty;
TUI `detach_document` no-ops when nothing is attached; migrate-module cluster (prefer a standalone
`<ts>` dir over a filename prefix; strip only the single ts component; `is_compact_utc` validates the
date). Documented/deferred (cosmetic or moderate-risk): highest-generation redundancy pick,
`mutate_categories`/`set_volume_max_size` save-failure rollback, `owner_value_summary` case-merge,
GUI vault-picker debounce, global compact/part flag-stripping, "between 1 and 0" message.

### Likely (split verdicts) — human-judgment pass
`append_frame` non-saturating `set_len` (now checked), stale-`.rekey` reopen, dry-run `MAX_PATH_LEN`
parity, merge equal-second tie, merge duplicate-id collapse. Treated as low/defense-in-depth; the
`set_len` one is folded into the Low batch.
