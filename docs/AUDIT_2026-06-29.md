# Deep Audit — 2026-06-29

A multi-agent, adversarially-verified bug hunt and security review across the **entire**
codebase (core, desktop, ffi), run in iterative rounds. Each candidate finding was confirmed
by ≥2 independent verifier agents under a stated threat model before being acted on.

**Threat model (security):** a LOCAL encrypted password/asset vault. In scope: theft of the
on-disk vault files (confidentiality + tamper/rollback detection), a crafted vault /
other-vault / document fed to open/merge/extract/migrate, malicious filenames/paths/content,
and other local processes/users. Out of scope: an attacker already running as the user with
the vault unlocked in RAM.

Result: **no Critical or High** findings. The vault's core cryptography, atomic-write
discipline, and input sanitization are strong. The actionable findings were a symlink-follow
gap on the export/extract *write* path, a CSV formula-injection gap, an O(N²) read scan, and
a few defense-in-depth items.

### Fixes applied this round

- **M1 — symlink-hardened export.** Added `vault::reject_symlinked_descendants(root, leaf)`
  and call it before `create_dir_all` in `export_document_into` (`vault.rs`) and `cli_extract`
  (`main.rs`). Any pre-planted symlink among the intermediate directories of the (user-chosen,
  reused) export root is now rejected — mirroring the import-side `reject_symlink_dir`
  discipline — so decrypted plaintext can't be redirected outside the root, and the
  `harden_dir` chmod no longer runs through an unverified intermediate. Regression test:
  `export_document_into_rejects_symlinked_intermediate`.
- **M2 — CSV anti-formula prefixing.** `csv::esc` now prefixes a cell that begins with
  `= + - @` with a single quote so spreadsheets render it as text. Realistic passwords (no
  leading trigger) round-trip unchanged. Test: `esc_neutralizes_leading_formula_triggers`.
- **M3 — O(1) `entry()`.** `Located` now carries `entry_index`; `reindex` populates it and
  `VolumeStore::entry` indexes directly (with a safe fallback scan on any desync), removing
  the per-read linear scan. Bulk read paths drop from O(N²) to O(N log N).
- **L2 — pre-sized Zeroizing read buffer.** `read_file_capped` now pre-sizes a `Zeroizing`
  buffer to the file's stat'd length (clamped to the ceiling) so `read_to_end` never
  reallocates and strands cleartext, and returns `Zeroizing<Vec<u8>>` directly.
- **Lock `O_NOFOLLOW`.** `WriteLock::acquire` now opens the lock file with `O_NOFOLLOW`
  (Unix), surfacing a symlinked lock path as `Locked` (via `ELOOP`) instead of following it —
  consistent with `single_instance.rs` and the volume writers; still no chmod on the lock.

### Planned (later rounds)
- **L1**: lower `MAX_RECOVERY_SALTS` and/or clamp recovery KDF params to a recovery floor.
- **L3**: enforce a per-partition entry-count ceiling in `load_manifest` and the import path.
- **Write-side O(N²)**: `staged_rewrite`/import re-serialize the whole partition manifest per
  `put`, so bulk *rewrites* remain O(N²) (availability-only, Ctrl-C-able). Batched-commit
  rewrite is a larger follow-up.

### Notable strengths (verified)
- `VolumeStore::read` cross-checks the AEAD frame's id, path, **and** decrypted size against
  the manifest (AAD binds only `vault_id+partition`); manifests are AEAD-authenticated before
  any `serde_json` parse.
- Rigorous symlink/path hardening almost everywhere (import rejects symlinked dirs; reads use
  `O_NOFOLLOW`/`symlink_metadata`; volume writes use `O_NOFOLLOW`+`create_new`/O_EXCL); M1 was
  the outlier, now closed.
- Conservative, audit-driven Argon2 bounds; deliberate secret zeroization on the write path
  (L2 was the one missed read path, now fixed); atomic rename-replace with parent fsync.
